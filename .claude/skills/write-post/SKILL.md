# 블로그 포스트 작성 스킬

아웃라인과 키워드를 받아 Hugo 마크다운 블로그 포스트를 작성합니다.

## 실행 절차

1. `prompts/content_system.txt`를 읽어서 문체/SEO/구조/차별화/금지 규칙을 로딩합니다.
2. `prompts/content_user.txt`를 읽어서 요구사항 템플릿을 확인합니다.
3. 아웃라인이 제공되지 않았다면, 먼저 `/outline` 스킬로 아웃라인을 생성하거나 사용자에게 요청합니다.
4. 기존 발행 글 목록에서 내부 링크 후보를 찾습니다 (`blog/content/posts/` 확인).
5. 모든 규칙을 준수하여 본문을 작성합니다.

## 입력

`$ARGUMENTS` — 키워드 (아웃라인은 대화 컨텍스트에서 참조하거나 별도 제공)

예시: `/write-post 전기차 보조금 2026`

## 핵심 체크포인트 (놓치지 말 것)

### 문체
- ~요체 사용
- AI스러운 표현 금지 ("물론이죠!", "~에 대해 알아볼까요?", "마무리하며")
- 본문 이모지 금지 (제목에만 1개)

### SEO
- 키워드 본문 3-5회 자연스럽게 삽입
- 첫 문단에 키워드 포함
- H2 태그 구분

### 구조
- 도입부: 첫 2문장 안에 핵심 팩트
- 요약 박스: 도입부 뒤 인용 블록(>)
- 본문: 각 섹션 200-400자
- FAQ + 마무리

### 차별화
- 비교표/계산 예시/체크리스트 등 독자적 가치 최소 1개
- 숫자 비교 주제에는 마크다운 표 필수

### 금액/수치
- 단위 명확히 표기
- 교차 검증 (단가 × 수량 = 총액)
- 표와 본문 수치 일치 확인

### 커버 이미지
- `premium_photo-` URL 사용 금지 (유료 이미지 → 404 발생)
- `photo-` 로 시작하는 무료 이미지만 사용
- URL을 추측/생성하지 말고, Unsplash 웹사이트에서 실제 이미지 URL 확인
- 이미지 선택 후 `curl -s -o /dev/null -w "%{http_code}" "<URL>"` 으로 HTTP 200 확인 필수
- 기존 포스트와 이미지 중복 확인
- `cover.alt` 필드에 한국어 대체 텍스트 필수 작성 (이미지 깨짐 대비 + SEO/접근성)

## 출력

Hugo 마크다운 형식 본문 (front matter 없이). 1500-2500자 분량.
