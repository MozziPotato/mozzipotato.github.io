## 트렌드 데이터 (네이버 데이터랩 + Google Trends 실측)
{trend_data}

위 데이터에서:
- naver_avg: 네이버 검색 트렌드 평균 (0-100, 높을수록 검색량 많음)
- google_avg: 구글 트렌드 평균 (0-100)
- trend: rising(상승)/stable(안정)/declining(하락)
- 데이터가 없는 키워드는 검색량이 매우 적은 것으로 판단하세요

## 평가 시 트렌드 데이터 활용 기준
- combined_score 50 이상: 검색량 충분
- combined_score 30-50: 검색량 보통, 콘텐츠 품질로 승부 가능
- combined_score 30 미만: 검색량 부족, 특별한 이유 없으면 제외
- trend가 "rising"인 키워드 우선 선정

## 평가 기준
1. 검색 의도: 정보성 검색 의도가 있는가? (순수 네비게이션 의도는 제외)
2. 콘텐츠 적합성: 1500자 이상의 유용한 글을 쓸 수 있는 주제인가?
3. 경쟁도: 대형 사이트가 아닌 개인 블로그가 노출될 수 있는가?
4. 구체성: 너무 넓지 않고 구체적인 롱테일 키워드인가?
5. 검색 의도 깊이: 정보를 찾는 구체적인 이유가 명확한가?
6. 트렌드 잠재력: 계절적이거나 현재 트렌드와 연관이 있는가?
7. 광고 수익성 (중요): 이 키워드에 광고비를 많이 쓰는 광고주가 있을 가능성이 높은가?
   - CPC가 높은 분야: 보험, 대출, 금융, 법률, 의료, 인테리어, 이사, 교육, 자격증, IT서비스 등
   - 상업적 정보 의도: 구매 결정 전 비교/추천/후기를 찾는 키워드 (예: "○○ 비교", "○○ 추천", "○○ 후기", "○○ 비용")
   - AdSense 광고 매칭: 해당 키워드로 검색했을 때 관련 광고가 노출될 가능성
   - 단순 정보성(예: "○○ 뜻")보다 소비/서비스와 연결되는 키워드를 우선 선정

## 출력 형식 (반드시 JSON 배열)
```json
[
  {
    "keyword": "키워드",
    "approved": true,
    "volume_hint": "medium",
    "competition_hint": "low",
    "cpc_hint": "medium",
    "reason": "선정/제외 이유 (광고 수익성 판단 근거 포함)",
    "intent_depth_hint": "deep/moderate/shallow"
  }
]
```

volume_hint, competition_hint, cpc_hint는 high/medium/low 중 하나로 추정해주세요.
트렌드 데이터가 있는 경우, 실측 데이터를 기반으로 volume_hint를 판단하세요.
cpc_hint 기준: high=광고주 지출이 큰 분야(금융/보험/법률/의료 등), medium=소비재/서비스 관련, low=광고 매칭이 어려운 순수 정보성 키워드.
